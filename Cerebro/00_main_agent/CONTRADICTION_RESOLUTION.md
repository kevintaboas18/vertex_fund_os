# Contradiction Resolution

Contradictions are information, not errors to smooth away.

## Common combinations

| Combination | Interpretation | Main-agent label |
|---|---|---|
| Strong business, weak technical | Quality may be intact; timing is unconfirmed | Quality watch / wait for confirmation |
| Weak business, strong technical | Price leadership without durable economics | Speculative momentum only |
| Strong valuation, weak technical | Possible value trap | Value watch |
| Expensive valuation, strong growth and technical | Premium still validated | Momentum candidate if gates pass |
| Strong total, low risk score | Aggregate hides survival risk | Apply risk override |
| DCF high, reverse DCF demanding | Model assumptions may be optimistic | Lower valuation confidence |

## Resolution rules

1. Keep each category score unchanged.
2. Identify the exact metric causing disagreement.
3. Prefer an explicit gate over narrative reconciliation.
4. If two agents use different definitions of the same input, resolve at the common-facts layer and rerun both.
5. If the contradiction is economic rather than data-driven, show both views and name the condition that would resolve it.
